Call Tool

Brief

Hey Dan Ryan,

Betsy and I are meeting with tech about some call tool UX improvements we’d like to consider. We’re adding functionality, and these will have to change eventually, but since we don’t know when the next updates will be, we want to make sure that our users are getting the most out of the tool, and that it fits appropriately with Dashboard.     Some of what we’re thinking:

*           “Resource page:” Although we want to give users the easiest pathway to a phone call (i.e. they get to the script page ASAP,) I think we need to develop something that they can click back to, where they can find the training videos, talking points, FAQ, etc.

*          Menu of Calls: as we develop more and more phonebanks (these Women for Obama event recruitment calls, for example,) we’re going to have more than one state wide call campaign. (Not unlike “rooms” we had discussed.) We should have these available for all users. A women who arrives at the call tool today will not know that she has the option to make Women for Obama calls. We need to give her that option.  I envision this on the resource page.   Scheduling this meeting for Monday at 3:30. Do you have anyone who should be there? Will include you in the calendar invite. Thank you very much.   Bridget

Notes

  • Critical Issues With Current Flow

    • Login screen displays for a few seconds then disappears regardless of use input or hesitation
    • If user tries to login at the screen, it will change regardless of whether they are successful during login
    • BO Login broken - Facebook successful
    • Branded login - currently, aside fro the URL, there are no visual cues that tell me I am at an official Obama site
    • Post creation of BO account, empty page is displayed.
    • If closing page and reopening, page is populated

  • Script If we want public users calling this, we can’t assume any competency in the UI or with speaking on cue with something they are not comfortable enough

    • Hello, can I speak with NAME / This should pull the callers actual name into the script. Would keep conversation flowing - the slight pause to glance back to the call box could feel awkward.
    • My name is, same as above - simply pull the User’s name in.

  • Core UI

    • Currently, there is only the current call. Other UI considerations should be:

      • Global header bar with:
        • resources & training
        • help - if something doesn’t work, what should the user do
        • home - not the script page, a “what is the call tool” or “how you can help OFA”
          • Even though the user is brought to the script immediately, it should be clear that this is not the “home” page of the call tool
        • On the screenshot below, “Not your state” and my account info are displayed without any CTA. These should not be directly displayed on the script because they don’t have a direct influence on the current call.
        • Call List
          • A recent call list should be available to a user in case they:
            • Skipped the page by accident
            • Thought of another note to add on the specific call later on
            • Lost the connection or wanted to retry in a few minutes
              • People will often say “XYZ is not here now, but she’ll be back in 20 minutes” - This is an open invite to call back and a missed opportunity with the current UI since the Caller can’t go back
          • Skipping a Call displays awkward “Person skipped” box - this provides no actionable items for the user. Did they skip by mistake? Do they need to add comments for why they skipped that call? Displaying this just for the sake of showing it is not the right reason.
        • Current Call has wasted space
          • Why show “If no” before the user clicks No? the box would be 50% smaller if that was an accordion or an ajax pull to display the different options ONLY if the user selects “no”
          • I was not comfortable contacting this voter isn’t a required field - I can simply “Skip this voter” without giving any reason - it should request something to be required or remove the options that could be assumed

    • Rooms

      • Women For Obama or others should be displayed under the global nav
        • You are currently calling the Women For Obama list.
        • Immediate nav item, possibly dropdown, to quickly access other lists
        • List order permanent or possibly ordered by use.
          • Enter Women for Obama room, call 4 people > Move to latinos room by mistake, call 0 > move to Undecided room, call 8 people.
          • This user’s list would show: Undecided, then Women for Obama, then potential room matches based on their previous session, and Latinos last. We want to keep there 0-call option available in case they left the room on error.
